## 点子管理规则

### 状态流转
```
新点子 → 需求确认 → 待办 → 进行中 → 测试中 → 已完成
                    ↓
                  搁置
```

### 优先级定义
- **P0**: 紧急重要，立即处理
- **P1**: 重要不紧急，近期安排
- **P2**: 一般重要，后续考虑
- **P3**: 低优先级，可能不做

### 更新规则
1. 每次讨论后更新相关点子
2. 状态变化时更新最后更新时间
3. 重大进展时记录详细日志
4. 搁置或取消时说明原因
