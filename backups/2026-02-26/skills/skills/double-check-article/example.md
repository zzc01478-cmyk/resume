# Double Check Article 使用示例

## 示例 1：基本使用

### 输入文章
```
人工智能正在快速发展，它已经在我们地生活中扮演着越来越重要的角色。从智能手机到自动驾驶汽车，人工智能技术正在改变我们的生活方式。然而，随着技术的进步，我们也需要关注人工智能带来的伦理和社会问题。
```

### 检查命令
```bash
./check_article.sh "人工智能正在快速发展，它已经在我们地生活中扮演着越来越重要的角色。从智能手机到自动驾驶汽车，人工智能技术正在改变我们的生活方式。然而，随着技术的进步，我们也需要关注人工智能带来的伦理和社会问题。"
```

### 预期输出
```
📊 文章检查开始
================================
📈 基本信息：
- 字符数: 120
- 行数: 1
- 词数: 30

🔍 错别字检查：
  - 注意 '地': 的用于形容词，得用于补语，地用于状语
    （发现："在我们地生活中" 应为 "在我们的生活中"）

🔤 标点符号检查：
  ✅ 标点符号使用基本规范

💬 表达清晰度检查：
  ✅ 句子长度适中

🎯 整体评估：
  文章质量评分: 8/10

💡 改进建议：
  ✅ 文章质量很好，建议修正错别字后发布
```

## 使用技巧

### 1. 批量检查多个文件
```bash
for file in *.txt; do
    echo "检查文件: $file"
    ./check_article.sh -f "$file"
    echo ""
done
```

### 2. 集成到编辑流程
```bash
# 在保存文件时自动检查
#!/bin/bash
FILE="$1"
if [[ "$FILE" == *.md ]] || [[ "$FILE" == *.txt ]]; then
    ./check_article.sh -f "$FILE"
fi
```

### 3. 生成检查报告
```bash
# 将检查结果保存到文件
./check_article.sh -f article.txt > check_report.txt
```

### 4. 实时检查
```bash
# 监控文件变化并自动检查
inotifywait -m -e close_write . | while read path action file; do
    if [[ "$file" == *.md ]]; then
        ./check_article.sh -f "$file"
    fi
done
```

## 自定义配置

### 1. 添加自定义错别字
编辑 `check_article.sh` 文件，在 `TYPO_DICT` 数组中添加：
```bash
TYPO_DICT+=(
  "您你:您表示尊敬，你表示一般"
  "份分:份表示数量单位，分表示分开"
)
```

### 2. 调整检查严格度
修改评分规则：
```bash
# 在 check_article 函数中调整扣分规则
if [ $found_typos -eq 1 ]; then
    score=$((score - 1))  # 改为扣1分
fi
```

### 3. 添加专业术语检查
```bash
# 添加专业术语检查函数
check_technical_terms() {
    local article="$1"
    local technical_terms=("API" "SDK" "框架" "部署")
    
    for term in "${technical_terms[@]}"; do
        if echo "$article" | grep -q "$term"; then
            echo "  - 使用专业术语: $term"
        fi
    done
}
```

## 注意事项

1. **不是万能的**：此工具只能检查常见问题，无法替代人工校对
2. **上下文理解有限**：对于需要上下文理解的错误可能无法识别
3. **持续改进**：错别字库需要定期更新和维护
4. **结合使用**：建议结合其他校对工具和人工检查
